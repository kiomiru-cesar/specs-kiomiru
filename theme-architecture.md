## 要点まとめ（まずここだけ読めばOK）

このドキュメントは、**テーマ構成（WordPress テーマのファイル設計・運用方針）の提案書**です。実案件で運用している構成をベースに、迷いにくく安全に拡張できる形を目指します。

### 設計の前提（大事な考え方）

- **入口は薄く**：`functions.php` は読み込み（require）中心にし、実装は `function/` 配下へ分割する。
- **責務で分割**：表側（theme）と管理画面（admin）を分離し、影響範囲を小さくする。
- **テンプレ階層を活かす**：`archive.php` / `single.php` は「親」として制御し、投稿タイプごとの差分は `include/` 側で吸収する。
- **ブロックは隔離**：ブロックエディタ（Lazy Blocks / Gutenberg）用の処理は `blocks/` に集約する。

### どこを編集すべきか（迷った時の判断）

| やりたいこと | 編集する場所 |
| :--- | :--- |
| 固定ページの専用デザイン | `template/` |
| 再利用する部品（カード・見出し・共通パーツ） | `include/` |
| 投稿タイプごとの一覧（アーカイブ）の見た目 | `include/archive/` |
| ブロックの表示 | `blocks/` |
| 表示に必要なデータ準備 | `function/theme/helper.php` |
| サイトの枠・共通出力 | `function/theme/layout.php` / `include/common/` |
| 管理画面の調整 | `function/admin/admin.php` |

## 1. ディレクトリ構成と役割

テーマ内の主要なディレクトリの役割です。役割が近いファイルが混在すると属人化しやすいため、「責務」でフォルダを分けています。

### template/（固定ページ用テンプレート）

WordPress 管理画面の「固定ページの属性 > テンプレート」で選択できるテンプレートが入っています。新しい固定ページのデザインを作る場合はここに追加します。

- 例：`page_sitemap.php` → サイトマップ用テンプレート

### include/（パーツ・部分テンプレート）

ページの一部として読み込まれる「部品」が入っています。

| サブディレクトリ | 役割 |
| :--- | :--- |
| **archive/** | 一覧ページでループ処理される「記事カード」部分（例：`news.php`, `event.php`） |
| **common/** | ヘッダー、フッター、SNS アイコンなど、全ページ共通の部品 |

### blocks/（Lazy Blocks 用テンプレート）

ブロックエディタ（Gutenberg）で使用するカスタムブロックの PHP ファイルが入っています。`lazyblock-` で始まるフォルダ内の `block.php` がブロックの表示内容を制御します。

- 例：`lazyblock-faq/block.php` → Q&A ブロックの表示

### acf-json/（ACF 設定ファイル）

プラグイン「Advanced Custom Fields」の設定が JSON 形式で自動保存される場所です。本番・検証環境間でカスタムフィールドの設定を簡単に同期できます（手動編集は不要）。

### function/（機能定義）

テーマの裏側で動く PHP 関数群です。`functions.php` から役割ごとに分割して読み込まれています。

> **提案方針**  
> `functions.php` は「入口（読み込み）」に留め、ロジックは `function/` 以下へ分割します。これにより、追加・修正時の影響範囲が明確になり、引き継ぎや AI 依頼でも事故が起きにくくなります。

#### function/theme/（サイトの見た目・動き）

| ファイル名 | 役割 | 具体的な中身の例 |
| :--- | :--- | :--- |
| **core.php** | サイトの基本設定 | アイキャッチ画像の有効化、不要なヘッダータグの削除、CSS/JS の読み込み設定など |
| **structure.php** | サイトの構造 | カスタム投稿タイプの登録、URL（スラッグ）の自動設定、一覧ページの表示件数変更など |
| **layout.php** | 外枠・共通パーツ | ヘッダーメニュー、フッター、パンくずリスト、ページネーションなど、サイトの「枠」を作る部分 |
| **helper.php** | データ準備（取得・加工） | ページタイトルの取得、パンくずデータの生成、本文の自動整形など、表示用の「素材」を用意する部分 |
| **block.php** | 表示補助（HTML 出力） | リンクや画像などを安全に出力するヘルパー関数群（`theme_block_link`, `theme_block_img` など）。データ取得・加工は `helper.php` 側で行う |
| **shortcodes.php** | プロジェクト用 | URL 取得などの汎用ショートコードや、特定の場所でしか使わない独自機能 |
| **assets.php** | ファイルパス管理 | 画像や CSS の場所（URL/パス）を間違いなく取得するための機能 |

#### function/admin/（管理画面・裏側）

| ファイル名 | 役割 | 具体的な中身の例 |
| :--- | :--- | :--- |
| **admin.php** | 管理画面カスタマイズ | 記事一覧への「URL 列」追加、ログイン画面のロゴ変更、CF7 のカスタマイズ、ブロックエディタの制限など |

## 2. 主要ファイル（テンプレート）の役割

どのページがどのファイルで表示されているかの対応です。

### front-page.php（トップページ）

サイトのトップページ（ホーム）を表示します。このテーマではトップの内容は管理画面のブロックエディタで編集でき、PHP 側はコンテンツを出力するだけのシンプルな構造です。

### archive.php（一覧ページ）

「お知らせ」や「イベント」などの記事一覧を表示する**親ファイル**です。URL に合わせて **`include/archive/投稿タイプ名.php`** を自動で読み込み、デザインを切り替えます。

- 例：お知らせ一覧 → `include/archive/news.php` が使われる

「親（archive.php）＋差分（include/archive）」の構造を基本とし、一覧の追加・改修は include 側で完結させます。

### single.php（詳細ページ）

記事の個別ページを表示します。「イベント」（`event`）の場合は専用デザイン（プロフィール表示など）が適用され、それ以外は標準的なブログ記事として表示される分岐になっています。

> **提案方針**  
> `single.php` は「親ファイル」として制御し、投稿タイプごとの差分（専用レイアウトや表示部品）は `include/single/投稿タイプ名.php` に切り出して管理します。

## 3. 運用・カスタマイズのポイント

### 編集時のルール

1. **入口を肥大化させない**  
   `functions.php` に長大な処理を書かず、`function/` 以下の適切なファイルに追記する（`functions.php` は読み込み中心）。
2. **責務で置き場所を決める**  
   変更内容に応じて、次のように使い分ける。

| 変更内容 | 編集する場所 |
| :--- | :--- |
| 一覧ページのデザイン | `include/archive/` |
| 詳細ページのデザイン | `include/single/` |
| 固定ページのデザイン | `template/` |
| 特定のブロックのデザイン | `blocks/` |
| 管理画面の調整 | `function/admin/admin.php` |

### ファイルの探し方（よくあるケース）

| やりたいこと | 編集するファイル |
| :--- | :--- |
| お知らせ一覧のデザインを変えたい | `include/archive/news.php` |
| トップページの内容を変えたい | 管理画面の固定ページ編集（または `front-page.php`） |
| サイトマップの構造を変えたい | `template/page_sitemap.php` |

## 4. AIアシスタントへの指示ルール

ChatGPT、Claude、GitHub Copilot などの AI にコード修正を依頼するときは、以下のテキストを一緒に渡すと、本テーマの設計思想に沿ったコードが返ってきやすくなります。

```markdown
# AIへの指示ルール (PHP/WordPress)
このテーマの設計思想に従ってコーディングしてください。

## 重視する制約
1. **ファイル責務の遵守**
   - HTML 出力に関わる処理は `layout.php` または `include` 内のテンプレートに記述する。
   - データの取得・加工ロジックは `helper.php` に記述する。
   - 管理画面のカスタマイズは `admin.php` に記述する。
   - サイト全体の初期設定やフックは `core.php` または `structure.php` に記述する。
   - ショートコード定義は `shortcodes.php` に記述する。
   - カスタムブロック（Lazy Blocks）のロジックは `blocks/` 配下に記述する。

2. **入口の保持**
   - `functions.php` は読み込み（require）中心にし、処理は `function/` 以下へ分割すること。

3. **安全性の確保**
   - 画面に出力する変数は必ずエスケープすること（`esc_html`, `esc_url`, `theme_block_attr` 等）。
   - `block.php` に定義されたヘルパー関数（`theme_block_link`, `theme_block_img`）を優先的に使用すること。

4. **型宣言**
   - `declare(strict_types=1);` を使用し、引数と戻り値には型指定（`string`, `int`, `void` 等）を行うこと。
   - 関数定義時は `if (!function_exists('...'))` でラップし、再定義エラーを防ぐこと。

5. **命名規則**
   - 独自の関数名には必ず `theme_` プレフィックスを付けること。
```
